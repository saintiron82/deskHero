# 몬스터 기획서 (Monster Design Document)

## 1. 개요 (Overview)
본 문서는 게임에 등장하는 몬스터의 **종류(Species)**, **속성(Attributes)**, 그리고 **작명(Naming)** 규칙을 정의합니다.
*   **총 종족 수**: 25종 (일반 20종 + 보스 5종)
*   **속성 체계**: 6대 속성 (무속성 + 화염/냉기/바람 + 신성/암흑)
*   **총 데이터 수**: 150개 (25종 × 6속성)

---

## 2. 6대 속성 시스템 (Attribute System)
각 속성은 고유한 색상 테마와 접두사/이름 규칙을 가집니다.

| 속성 (Attribute) | 테마 색상 (Theme Color) | 느낌/키워드 | 비고 |
| :--- | :--- | :--- | :--- |
| **무속성 (Normal)** | **Green / Brown** | 숲, 동굴, 야수, 물리 | 원본(Original) 형태 |
| **화염 (Fire)** | **Red / Orange** | 불, 용암, 폭발, 열기 | 공격적, 파괴적 |
| **냉기 (Ice)** | **Blue / Cyan** | 얼음, 서리, 물, 수정 | 차가움, 경직 |
| **바람 (Wind)** | **Yellow / Green** | 번개, 태풍, 모래, 속도 | 빠름, 회피 |
| **신성 (Holy)** | **White / Gold** | 빛, 황금, 치유, 고대 | 신비로움, 정화 |
| **암흑 (Dark)** | **Purple / Black** | 어둠, 저주, 맹독, 그림자 | 공포, 타락 |

## 4. 데이터 구현 계획
1.  **이미지 생성**: `Raw_Green` 폴더에 `_A` ~ `_E` 접미사로 저장 (물리적 파일).
2.  **데이터 매핑**: `CharacterData.json`에서 위 표의 **'이름'** 을 사용하여 각 파일(`_A`~`_E`)에 매핑.
    *   예: `monster_slime_A` -> "숲 슬라임"
    *   예: `monster_slime_B` -> "마그마 슬라임" (색상 매칭 후 결정)
