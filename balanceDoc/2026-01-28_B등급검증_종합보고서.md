## 결론

**B 등급 재현 실패**

5회 테스트에서 B 등급이 한 번도 나오지 않았습니다.

이는 이전 자동 최적화(report_2026-01-28_auto-optimization.md)에서 기록된 B 등급이 **확률적 결과**였음을 시사합니다.

**원인 분석:**
1. GA(유전 알고리즘)의 무작위성으로 인해 매 실행마다 다른 결과
2. 히스토리 기반 focus stat 선택이 결과에 영향
3. Grid 탐색 후 RANDOM으로 선택되는 스탯이 달라짐

## 발견 사항

### 1. 일관성 문제

같은 설정으로 5회 테스트했으나 등급이 D~C 사이에서 변동:
- 시뮬레이션 분석기의 결과가 확률적임
- B 등급 달성은 "행운"의 요소가 있음

### 2. 공통 패턴

5회 테스트에서 공통적으로:
- **utility 카테고리 0% 사용** (time_extend, upgrade_discount)
- **starting_bonus 카테고리 0% 사용**
- **base_stats + currency_bonus 지배**

### 3. 과다 사용 스탯 (5회 공통)

- attack_percent
- multi_hit
- gold_multi_perm

### 4. 과소 사용 스탯 (5회 공통)

- time_extend
- upgrade_discount
- 모든 starting_bonus 계열

## 권장 조치

### 단기 (안정적 C 등급 유지)

현재 설정으로 C 등급은 안정적으로 달성 가능. 변경 불필요.

### 중기 (B 등급 안정화 시도)

1. **utility 카테고리 대폭 버프 필요**
   - time_extend: effect_per_level 0.5 → 2.0
   - upgrade_discount: effect_per_level 2.0 → 5.0

2. **starting_bonus 버프 또는 별도 분리**
   - 현재 설정으로는 사용 가치 없음

### 장기 (구조적 개선)

자동 최적화 보고서(report_2026-01-28_auto-optimization.md)의 "핵심 발견" 섹션 참조:
- Starting Bonus의 구조적 한계
- 다양성 0.50+ 달성의 어려움
