# 🤝 Advanced Agent Collaboration Protocol v2.0

## 🎯 개요 (Overview)
jina와 lily 간의 **지능형 협업 시스템**으로, 예측적 작업 분배, 자동 품질 보증, 그리고 실시간 컨텍스트 인식을 통해 최적의 개발 경험을 제공합니다.

## 🧠 지능형 역할 전환 시스템 (Intelligent Role Switching)

### 🔄 컨텍스트 인식 자동 전환 (Context-Aware Auto-Switch)
```yaml
고급_전환_로직:
  파일_기반_전환:
    기획_영역_jina:
      - docs/**/*.md (기획문서)
      - config/GameData.json (밸런스)
      - *.requirements.md (요구사항)
      - *.spec.md (스펙문서)

    구현_영역_lily:
      - src/**/*.cs (소스코드)
      - *.xaml (UI정의)
      - *.csproj (프로젝트파일)
      - tests/**/*.cs (테스트코드)

  키워드_기반_지능_전환:
    jina_활성화_신호:
      - "기획", "설계", "컨셉", "밸런스", "UX"
      - "요구사항", "스펙", "사용자경험"
      - "A/B테스트", "메트릭", "KPI"

    lily_활성화_신호:
      - "구현", "코드", "버그", "성능", "리팩토링"
      - "빌드", "테스트", "배포", "최적화"
      - "아키텍처", "보안", "API"

  상황_기반_예측_전환:
    기획_후_구현_전환: # jina → lily
      - 스펙_완성도 > 90% 시 자동 lily 준비모드
      - 구현_요청_키워드 감지 시 즉시 전환

    구현_문제_기획_전환: # lily → jina
      - 기술적_제약_발견 시 자동 jina 호출
      - 요구사항_모호함_감지 시 즉시 전환
```
