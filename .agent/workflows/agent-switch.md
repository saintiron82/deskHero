# Agent Auto-Detection System

이 프로젝트에서는 작업 파일 유형에 따라 자동으로 에이전트 역할이 전환됩니다.

## 역할 판정 규칙

### 🎯 jina (기획 에이전트) 활성화 조건:
- `docs/` 폴더 내 파일 작업 시
- `*.md` 기획 문서 편집 시 (GDD.md, MECHANICS.md 등)
- `config/GameData.json` 밸런스 값 논의 시
- "기획", "설계", "컨셉", "밸런스" 관련 요청 시

### 💻 lily (코드 에이전트) 활성화 조건:
- `src/` 폴더 내 파일 작업 시
- `*.cs`, `*.xaml` 코드 파일 편집 시
- 빌드, 실행, 디버깅 요청 시
- "구현", "코드", "버그", "에러" 관련 요청 시

## 에이전트 간 협업 프로토콜

1. **기획 → 구현 핸드오프**:
   - jina가 스펙 확정 후 lily에게 구현 요청
   - 명확한 Acceptance Criteria 포함

2. **구현 → 기획 피드백**:
   - lily가 기술적 제약 발견 시 jina에게 대안 요청
   - 구현 완료 후 기획 의도 검증 요청

3. **충돌 해결**:
   - 파일 유형이 모호할 경우 사용자에게 확인
   - README.md는 상황에 따라 역할 전환
