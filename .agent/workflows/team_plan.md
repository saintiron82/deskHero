# Team Planning Mode

이 워크플로우는 사용자의 요청에 대해 Jina(기획)와 Lily(구현)가 함께 논의하고, 보고서를 작성하여 승인을 받는 과정을 자동화합니다.

## 실행 절차

// turbo-all

1. **에이전트 정의 및 프로토콜 로드**:
   `view_file` 도구를 사용하여 다음 파일들을 반드시 읽습니다:
   - `.agent/jina.md`
   - `.agent/lily.md`
   - `.agent/workflows/advanced-collaboration.md`

2. **합동 분석 및 토론 (Strict Constraints 적용)**:
   사용자의 요청을 바탕으로 두 에이전트가 관점을 교환합니다. 단, 다음 **금지 사항**을 엄격히 준수합니다:
   *   🚫 **의미 없는 논쟁 금지**: 1턴 이상 같은 주장을 반복하지 않는다. 이견이 좁혀지지 않으면 Jina(기획)의 결정을 따른다.
   *   🚫 **오버엔지니어링 금지**: "혹시 필요할지도 모르는" 기능은 모두 배제한다. 현재 요구사항에 필요한 최소한의 구현(MVP)에 집중한다.
   *   🚫 **기능 비대화(Feature Creep) 금지**: 사용자가 요청하지 않은 '보너스 기능'을 제안하지 않는다.
   *   🚫 **반복 대화 차단**: 서로 "동의합니다", "좋은 생각입니다" 등의 의미 없는 칭찬으로 턴을 낭비하지 않는다. 바로 본론만 말한다.

3. **기획 및 승인 요청서 작성**:
   `advanced-collaboration.md`의 핸드오프 체크리스트를 참고하여 `implementation_plan.md` 포맷으로 정리합니다.
   *   **목표**: 무엇을 만들 것인가?
   *   **상세 스펙**: 어떻게 동작하는가?
   *   **기술적 접근**: 어떻게 구현할 것인가?
   *   **예상 위험**: 무엇을 조심해야 하는가?

4. **사용자 승인 대기 (STOP)**:
   `notify_user`를 호출하여 기획안을 제시하고, **명시적인 승인**을 요청합니다.
   *   주석: "승인되지 않으면 절대 구현 단계로 넘어가지 마시오."
