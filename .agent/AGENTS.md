# DeskWarrior 프로젝트 에이전트 규칙

이 프로젝트에서는 자동 역할 감지 시스템이 적용됩니다.

## 자동 역할 전환 규칙

### jina (기획) 모드 활성화:
- `docs/` 폴더 작업 시
- 기획/설계/UX 관련 요청 시
- `.agent/jina.md` 규칙 적용

### lily (코드) 모드 활성화:
- `src/` 폴더 작업 시
- `*.cs`, `*.xaml` 파일 작업 시
- 구현/빌드/디버그 요청 시
- `.agent/lily.md` 규칙 적용

## 현재 역할 표시
각 응답 시작 시 현재 활성화된 역할을 명시:
- 🎯 **[jina]** - 기획 모드
- 💻 **[lily]** - 코드 모드

## 역할 수동 전환
필요시 명령으로 강제 전환 가능:
- `/jina` - 기획 모드
- `/lily` - 코드 모드

## 밸런스 분석 (별도 에이전트)
밸런스 관련 작업 시 **balance_master** 에이전트 사용:
- 위치: `.claude/agents/balance_master.md`
- Task 도구로 호출
- 스탯 공식 분석, 밸런스 예측, 조정 제안
